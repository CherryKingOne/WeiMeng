WeiMeng/backend 模型供应商（providers）实现说明

更新时间：2026-03-12
状态：已落地，支持多供应商 + openai-compatible 扩展配置

一、模块定位
- 模块路径：`src/modules/providers`
- 目标：统一管理模型供应商配置、模型列表查询、系统模型配置与聊天调用。
- 当前供应商枚举：`openai`、`openai-compatible`、`qwen`、`volcengine`、`grok`、`gemini`、`anthropic`、`kimi`、`glm`、`minimax`、`deepseek`。
- 鉴权：`/api/v1/models/**` 全路由统一挂载 `Security(get_current_user_id)`。

二、关键实现约束
1. Base URL 与默认模型
- 标准供应商适配器在代码内定义 `DEFAULT_BASE_URL` 与 `DEFAULT_MODEL`。
- 标准供应商不通过 `.env` 注入 `BASE_URL`。

2. API Key 与配置来源
- 标准供应商 key 通过 `POST /api/v1/models/providers` 写入 `providers` 表。
- key 与 `user_id` 绑定，按用户隔离。
- `api_key` 使用加密存储，读取时解密到运行时内存（`provider_key_store`）。

3. openai-compatible 专项
- 通过 `/api/v1/models/providers/openai-compatible` 系列接口管理（增删改查）。
- 使用同一 `providers` 表存储，但采用合成 provider key（`openai-compatible:<hash>`）按模型隔离。
- 每条配置包含：`base_url`、`api_key`、`model`、`max_token`、`temperature`。

4. 系统模型配置
- 接口：`POST /api/v1/models/system`、`GET /api/v1/models/system`。
- 写入时要求 `type`（`text|image|video`）+ `provider` + `model_name`。
- 当前读取接口返回的是 `type=text` 的系统模型配置。

三、API（`src/modules/providers/api/router.py`）
统一前缀：`/api/v1/models`

- `GET /api/v1/models`
  - 查询供应商模型列表，支持 `provider` / `model` 参数。
  - 当传 `model` 时必须同时传 `provider`，否则 422。
  - `provider=openai-compatible` 时返回该用户配置的兼容模型列表。
- `GET /api/v1/models/{model_id}`
  - 模型详情查询，必须带 `provider` 查询参数。
- `POST /api/v1/models/chat`
  - 聊天调用入口；`message` 必填。
  - `provider`、`model_name` 可选：不传时回落到系统文本模型。
  - `stream=true` 返回 SSE（`text/event-stream`）。
  - qwen 支持：`enable_thinking`、`thinking_budget`。
  - volcengine/deepseek 支持：`thinking` 对象（深度思考参数规则按 provider 校验）。
- `POST /api/v1/models/providers`
  - 标准供应商 key 配置（不接受 `provider=openai-compatible`）。
- `POST /api/v1/models/providers/openai-compatible`
  - 新增一条兼容模型配置。
- `GET /api/v1/models/providers/openai-compatible`
  - 列表查询兼容模型配置。
- `PUT /api/v1/models/providers/openai-compatible`
  - 按 `model` 更新兼容模型配置。
- `DELETE /api/v1/models/providers/openai-compatible?model=...`
  - 删除兼容模型配置。
- `POST /api/v1/models/system`
  - 配置系统模型，请求体示例：`{"type":"text","provider":"qwen","model_name":"qwen-max"}`。
- `GET /api/v1/models/system`
  - 获取当前用户系统文本模型配置状态。
- `GET /api/v1/models/providers`
  - 返回标准供应商能力与配置状态（该接口不返回 openai-compatible 的细分模型项）。

四、聊天参数（`POST /api/v1/models/chat`）
- `provider`（可选）：不传则走系统模型；可传 `openai-compatible`。
- `model_name`（可选）：不传则使用供应商默认模型或兼容配置的首个模型。
- `message`（必填）：对话输入。
- `stream`（可选）：默认 `false`。
- `conversation_template`（可选）：覆盖默认模板。
- `temperature`、`max_tokens`（可选）。
- qwen 专属：
  - `enable_thinking`（可选）
  - `thinking_budget`（可选，`1~65536`）
- volcengine/deepseek 专属：
  - `thinking.type`：`enabled|disabled|auto`（deepseek 不支持 `auto`）
  - `thinking.reasoning_effort`：仅 volcengine 支持
