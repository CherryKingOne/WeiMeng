

# 剧本库管理系统 API 文档

## 1. 概述

本文档定义了剧本库管理的后端接口。系统后端采用 **PostgreSQL** 存储元数据，**RustFS** 负责底层文件存储。

*   **Base URL**: `http://<host>/v1`
*   **资源路径**: `/script-libraries` (对应业务中的“剧本库”)
*   **数据流逻辑**:
    1.  **PostgreSQL**: 负责生成并存储唯一的 `uuid`、`name`、元数据（创建时间、文件数量等）。
    2.  **RustFS**: 系统初始化时需创建一个名为 `Script Library` 的 Bucket（桶）。创建剧本库时，在此 Bucket 下创建以“剧本库名称”命名的目录/前缀。

## 2. 业务状态码大全

所有接口返回的 JSON 结构中均包含 `code` 字段，前端需根据此状态码进行业务逻辑判断。

| 状态码 (code) | 状态描述 (msg) | 含义说明 |
| :--- | :--- | :--- |
| **10000** | **Success** | **请求成功** |
| **20001** | Invalid Parameters | 请求参数错误（必填项缺失或格式错误） |
| **30001** | Database Error | 数据库操作失败（PG 连接或写入错误） |
| **30002** | RustFS Storage Error | RustFS 存储服务异常（桶不存在或网络错误） |
| **40001** | Library Name Exists | 剧本库名称已存在（名称需唯一） |
| **40002** | Library Not Found | 指定的剧本库 ID 不存在 |
| **40003** | Library Not Empty | 剧本库下包含文件，禁止直接删除（视业务规则而定） |
| **50000** | Internal Server Error | 服务器内部未知错误 |

## 3. 接口详情

### 3.1 创建剧本库

在数据库生成记录，并同步在 RustFS 的 `Script Library` 桶下初始化对应的目录结构。

*   **接口地址**: `/script-libraries`
*   **请求方法**: `POST`
*   **Content-Type**: `application/json`

#### 请求参数 (Body)

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| name | string | **是** | 剧本库名称（需全局唯一，用于 RustFS 目录名） |

#### 响应参数 (Data)

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | string | 剧本库唯一标识 (UUID) |
| name | string | 剧本库名称 |
| file_count | integer | 当前文件数量 (初始为 0) |
| created_at | string | 创建时间 (ISO 8601 格式) |

#### 示例代码

**Request:**

```json
POST /v1/script-libraries HTTP/1.1
Host: api.example.com
Content-Type: application/json

{
    "name": "悬疑推理剧本合集"
}
```

**Response (Success):**

```json
{
    "code": 10000,
    "msg": "Success",
    "data": {
        "id": "a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11",
        "name": "悬疑推理剧本合集",
        "file_count": 0,
        "created_at": "2023-10-27T10:00:00Z"
    }
}
```

**Response (Error - Name Exists):**

```json
{
    "code": 40001,
    "msg": "Library Name Exists",
    "data": null
}
```

### 3.2 删除剧本库

删除数据库中的元数据记录，并清理 RustFS 中对应的目录（注意：通常需确保库内无文件或执行级联删除）。

*   **接口地址**: `/script-libraries/{id}`
*   **请求方法**: `DELETE`

#### 请求参数 (Path)

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | string | **是** | 剧本库的 UUID |

#### 响应参数

无特定 Data 数据，仅返回状态。

#### 示例代码

**Request:**

```http
DELETE /v1/script-libraries/a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11 HTTP/1.1
Host: api.example.com
```

**Response (Success):**

```json
{
    "code": 10000,
    "msg": "Success",
    "data": null
}
```

**Response (Error - Not Found):**

```json
{
    "code": 40002,
    "msg": "Library Not Found",
    "data": null
}
```

---

### 3.3 获取剧本库列表

分页获取剧本库列表，包含元数据统计信息（如文件数量）。

*   **接口地址**: `/script-libraries`
*   **请求方法**: `GET`

#### 请求参数 (Query)

| 字段名 | 类型 | 必填 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- | :--- |
| page | integer | 否 | 页码 | 1 |
| page_size | integer | 否 | 每页数量 | 20 |
| keyword | string | 否 | 搜索关键字（按名称模糊搜索） | null |

#### 响应参数 (Data)

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| list | array | 剧本库列表数组 |
| └ id | string | 剧本库 UUID |
| └ name | string | 剧本库名称 |
| └ file_count | integer | 库内文件总数 (从 PG 读取) |
| └ created_at | string | 创建时间 |
| total | integer | 总记录数 |
| page | integer | 当前页码 |
| page_size | integer | 当前每页数量 |

#### 示例代码

**Request:**

```http
GET /v1/script-libraries?page=1&page_size=10 HTTP/1.1
Host: api.example.com
```

**Response (Success):**

```json
{
    "code": 10000,
    "msg": "Success",
    "data": {
        "list": [
            {
                "id": "a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11",
                "name": "悬疑推理剧本合集",
                "file_count": 5,
                "created_at": "2023-10-27T10:00:00Z"
            },
            {
                "id": "b1ffcd88-1d2a-4ef8-cc7e-7cc0bd490b22",
                "name": "古装情感本",
                "file_count": 12,
                "created_at": "2023-10-26T14:30:00Z"
            }
        ],
        "total": 2,
        "page": 1,
        "page_size": 10
    }
```

## 4. 数据库与存储设计说明

为了满足开发需求，后端实现时需遵循以下逻辑：

### PostgreSQL 表结构 (Schema 建议)

```sql
CREATE TABLE script_libraries (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(), -- 对应唯一UUID
    name VARCHAR(255) NOT NULL UNIQUE,             -- 剧本库名称，需唯一
    file_count INTEGER DEFAULT 0,                  -- 文件数量缓存
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 对应的文件表（供参考，用于上传文件逻辑）

--    id UUID PRIMARY KEY,
--    library_id UUID REFERENCES script_libraries(id),
--    file_path VARCHAR(500), -- 存储在RustFS中的完整路径
--    ...
-- );
```

### RustFS 存储逻辑

1.  **Bucket**: 全局统一使用 `Script Library` 桶。
2.  **Path**:
    *   当调用接口 **3.1** (创建) 时：在 RustFS 桶内逻辑上预留 `/{name}/` 路径。
    *   当后续上传文件时：文件实际存储路径为 `s3://Script Library/{Library Name}/{Filename}`。
3.  **Consistency**: 修改剧本库名称时，需同步修改 PG 记录和 RustFS 中的目录名称（这是一个耗时操作，建议禁止重命名或异步处理）。
