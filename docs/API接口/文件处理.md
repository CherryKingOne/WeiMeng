
# 文件管理系统 API 接口文档

**版本**: 1.0.0  
**基础 URL**: `http://{your-server-domain}/v1/file`  
**数据存储架构**:
- **PostgreSQL**: 存储文件元数据（UUID、文件名、大小、MIME类型、上传时间等）以及与剧本库（Knowledge Base）的关联关系。
- **RustFS**: 对象存储层。物理存储路径结构为 `/{bucket_name}/{library_name}/{filename}`。

## 1. 业务状态码大全 (Business Status Codes)

所有接口响应均包含 `code`, `message`, `data` 字段。以下是 `code` 的详细定义：

| 状态码 (code) | 含义 | 说明 |
| :--- | :--- | :--- |
| **200** | **成功** | 请求成功处理 |
| **400** | **请求参数错误** | 必填项缺失或参数格式不合法 |
| **401** | **未授权** | 用户未登录或Token过期 |
| **403** | **禁止访问** | 用户无权限操作该资源 |
| **500** | **服务器内部错误** | 未知系统异常 |
| **10001** | **文件上传失败** | 文件写入RustFS失败或IO错误 |
| **10002** | **文件大小超限** | 上传文件超过系统规定大小 |
| **10003** | **不支持的文件类型** | 文件后缀名不在允许范围内 |
| **10004** | **文件不存在** | 根据UUID在数据库中未找到记录 |
| **10005** | **文件删除失败** | 从RustFS或数据库删除时发生错误 |
| **10006** | **剧本库不存在** | 指定的剧本库名称无效 |
| **10007** | **存储桶不存在** | 指定的Bucket名称无效 |

## 2. 接口详情

### 2.1 上传文件接口

该接口用于将文件上传至 RustFS 存储系统，并在 PostgreSQL 中生成对应的元数据记录。
系统会根据传入的 `bucket_name`（桶名）和 `library_name`（剧本库名）在 RustFS 中自动创建或匹配文件夹结构。

- **请求方法**: `POST`
- **请求路径**: `/upload`
- **Content-Type**: `multipart/form-data`

#### 请求参数 (Request Parameters)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
| :--- | :--- | :--- | :--- | :--- |
| file | File | **是** | 二进制文件对象 | `script.pdf` |
| bucket_name | String | **是** | 存储桶名称 (如：吃饭) | `eating` |
| library_name | String | **是** | 剧本知识库名称 (如：早饭)，对应RustFS中的文件夹 | `breakfast` |
| description | String | 否 | 文件描述 | `早餐食谱V1` |

#### 响应参数 (Response Parameters)

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| file_id | String | 文件的唯一标识 (UUID)，由PG生成 |
| file_name | String | 原始文件名 |
| file_path | String | RustFS中的完整存储路径 |
| file_size | Number | 文件大小 (Bytes) |
| upload_time | String | 上传时间 (ISO 8601) |

#### 示例代码 (Example)

```bash
curl -X POST "http://localhost:8080/v1/file/upload" \
  -H "Content-Type: multipart/form-data" \
  -F "file=@/path/to/local/script.pdf" \
  -F "bucket_name=eating" \
  -F "library_name=breakfast" \
  -F "description=第一版早餐剧本"
```

#### 响应示例

```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "file_id": "a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11",
    "file_name": "script.pdf",
    "file_path": "/eating/breakfast/script.pdf",
    "file_size": 102405,
    "upload_time": "2023-10-27T10:00:00Z"
  }
}
```

### 2.2 删除文件接口

该接口用于删除指定的文件。
**业务逻辑**：
1. 根据 `file_id` 在 PostgreSQL 查找文件元数据，获取其在 RustFS 中的存储路径。
2. 调用 RustFS 接口删除物理文件。
3. 删除 PostgreSQL 中的元数据记录。
*(注：若需删除整个剧本库文件夹，建议增加专门的删除库接口，本接口仅针对单文件)*

- **请求方法**: `DELETE`
- **请求路径**: `/delete`
- **Content-Type**: `application/json`

#### 请求参数 (Request Parameters)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
| :--- | :--- | :--- | :--- | :--- |
| file_id | String | **是** | 文件的唯一标识 UUID | `a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11` |

#### 响应参数 (Response Parameters)

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| file_id | String | 被删除的文件ID |

#### 示例代码 (Example)

```bash
curl -X DELETE "http://localhost:8080/v1/file/delete" \
  -H "Content-Type: application/json" \
  -d '{
    "file_id": "a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11"
  }'
```

#### 响应示例

```json
{
  "code": 200,
  "message": "删除成功",
  "data": {
    "file_id": "a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11"
  }
}
```

### 2.3 获取指定剧本库的文件列表

该接口用于分页获取指定桶（Bucket）下某个剧本库（Library）中的所有文件列表。数据来源于 PostgreSQL 数据库查询。

- **请求方法**: `GET`
- **请求路径**: `/list`
- **Content-Type**: `application/x-www-form-urlencoded`

#### 请求参数 (Request Parameters)

| 参数名 | 类型 | 必填 | 说明 | 示例 |
| :--- | :--- | :--- | :--- | :--- |
| bucket_name | String | **是** | 存储桶名称 | `eating` |
| library_name | String | **是** | 剧本知识库名称 | `breakfast` |
| page | Number | 否 | 页码，默认为 1 | `1` |
| page_size | Number | 否 | 每页数量，默认为 20 | `10` |
| keyword | String | 否 | 文件名模糊搜索关键字 | `script` |

#### 响应参数 (Response Parameters)

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| total | Number | 总记录数 |
| page | Number | 当前页码 |
| list | Array | 文件列表数据 |
| └ file_id | String | 文件UUID |
| └ file_name | String | 文件名 |
| └ library_name | String | 所属剧本库 |
| └ file_size | Number | 文件大小(Bytes) |
| └ content_type | String | MIME类型 |
| └ upload_time | String | 上传时间 |

#### 示例代码 (Example)

```bash
curl -X GET "http://localhost:8080/v1/file/list?bucket_name=eating&library_name=breakfast&page=1&page_size=10"
```

#### 响应示例

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "total": 5,
    "page": 1,
    "list": [
      {
        "file_id": "a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11",
        "file_name": "script.pdf",
        "library_name": "breakfast",
        "file_size": 102405,
        "content_type": "application/pdf",
        "upload_time": "2023-10-27T10:00:00Z"
      },
      {
        "file_id": "b1ffcc88-1d2a-4bc9-aa5e-5cc8ad270b22",
        "file_name": "image.png",
        "library_name": "breakfast",
        "file_size": 20480,
        "content_type": "image/png",
        "upload_time": "2023-10-27T10:05:00Z"
      }
    ]
  }
}
```
