
# 用户认证系统 API 接口文档

**版本**: v1.0.0  
**基础路径 (Base URL)**: `/v1/auth`  
**数据库**: PostgreSQL  
**说明**: 本文档描述了用户注册、登录、密码重置及验证码发送的接口规范。

## 1. 全局说明

### 1.1 通用响应结构
所有接口均返回 JSON 格式数据，结构如下：
```json
{
  "code": 200,        // 业务状态码
  "message": "success", // 提示信息
  "data": { ... }     // 业务数据
}
```

### 1.2 业务状态码大全
| 状态码 (code) | 含义 | 说明 |
| :--- | :--- | :--- |
| **200** | 操作成功 | 请求处理成功 |
| **400** | 请求参数错误 | 参数缺失或格式不正确 |
| **401** | 未授权 | Token 无效或过期 |
| **500** | 服务器内部错误 | 系统异常 |
| **10001** | 邮箱格式错误 | 邮箱不符合标准格式 |
| **10002** | 验证码发送失败 | 邮件服务异常 |
| **10003** | 验证码错误 | 验证码不匹配 |
| **10004** | 验证码已过期 | 验证码超过5分钟有效期 |
| **10005** | 邮箱已注册 | 注册时该邮箱已存在 |
| **10006** | 用户不存在 | 登录或重置密码时邮箱未注册 |
| **10007** | 密码错误 | 登录密码不匹配 |
| **10008** | 操作过于频繁 | (虽然本系统无限制，保留此码以备后续拓展) |

## 2. 接口详情

### 2.1 发送邮箱验证码

*   **接口描述**: 向指定邮箱发送6位数字验证码。
*   **请求方法**: `POST`
*   **请求路径**: `/send-code`
*   **规则说明**:
    *   验证码有效期为 **5分钟**。
    *   **无发送频率限制** (No Rate Limit)。

#### 请求参数 (Body - JSON)
| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| email | string | **是** | 目标邮箱地址 |
| type | string | **是** | 业务类型：`register` (注册), `reset` (重置密码) |

#### 响应参数 (Data)
*无特定返回数据，仅返回成功状态。*

#### 示例代码
```bash
curl -X POST "http://api.example.com/v1/auth/send-code" \
     -H "Content-Type: application/json" \
     -d '{
           "email": "user@example.com",
           "type": "register"
         }'
```

#### 返回示例
```json
{
  "code": 200,
  "message": "验证码发送成功",
  "data": null
}
```

### 2.2 用户注册

*   **接口描述**: 使用邮箱、验证码和密码完成新用户注册。
*   **请求方法**: `POST`
*   **请求路径**: `/register`
*   **数据库行为**: 向 PostgreSQL `users` 表插入新记录，密码需加密存储 (如 bcrypt)。

#### 请求参数 (Body - JSON)
| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| email | string | **是** | 邮箱地址 |
| code | string | **是** | 6位数字验证码 |
| password | string | **是** | 设置的登录密码 (建议前端传输前Hash或HTTPS传输) |

#### 响应参数 (Data)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| user_id | string | 用户唯一标识 (UUID) |
| email | string | 注册邮箱 |
| created_at | string | 注册时间 (ISO 8601) |

#### 示例代码
```bash
curl -X POST "http://api.example.com/v1/auth/register" \
     -H "Content-Type: application/json" \
     -d '{
           "email": "user@example.com",
           "code": "123456",
           "password": "MySecretPassword123"
         }'
```

#### 返回示例
**成功：**
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "user_id": "a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11",
    "email": "user@example.com",
    "created_at": "2023-10-27T10:00:00Z"
  }
}
```
**失败 (验证码过期)：**
```json
{
  "code": 10004,
  "message": "验证码已过期，请重新获取",
  "data": null
}
```

### 2.3 用户登录

*   **接口描述**: 用户使用邮箱和密码进行登录，获取访问令牌。
*   **请求方法**: `POST`
*   **请求路径**: `/login`

#### 请求参数 (Body - JSON)
| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| email | string | **是** | 注册邮箱 |
| password | string | **是** | 登录密码 |

#### 响应参数 (Data)
| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| token | string | 鉴权 Token (如 JWT) |
| user_info | object | 用户基本信息 |
| └ id | string | 用户ID |
| └ email | string | 用户邮箱 |

#### 示例代码
```bash
curl -X POST "http://api.example.com/v1/auth/login" \
     -H "Content-Type: application/json" \
     -d '{
           "email": "user@example.com",
           "password": "MySecretPassword123"
         }'
```

#### 返回示例
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user_info": {
        "id": "a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11",
        "email": "user@example.com"
    }
  }
}
```

### 2.4 忘记密码 (重置密码)

*   **接口描述**: 用户忘记密码后，通过邮箱验证码设置新密码。
*   **请求方法**: `POST`
*   **请求路径**: `/reset-password`
*   **数据库行为**: 更新 PostgreSQL `users` 表中对应用户的 `password_hash` 字段。

#### 请求参数 (Body - JSON)
| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| email | string | **是** | 注册邮箱 |
| code | string | **是** | 6位数字验证码 |
| new_password | string | **是** | 新的登录密码 |

#### 响应参数 (Data)
*无特定返回数据，仅返回成功状态。*

#### 示例代码
```bash
curl -X POST "http://api.example.com/v1/auth/reset-password" \
     -H "Content-Type: application/json" \
     -d '{
           "email": "user@example.com",
           "code": "882910",
           "new_password": "NewStrongPassword!@#"
         }'
```

#### 返回示例
```json
{
  "code": 200,
  "message": "密码重置成功，请使用新密码登录",
  "data": null
}
```

## 附录：PostgreSQL 数据表设计建议

虽然 API 接口对外隐藏了数据库细节，但为了后端开发参考，建议的数据表结构如下：

**1. users 表 (用户表)**
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

**2. verification_codes 表 (验证码表)**
*注：验证码也可以存储在 Redis 中，若必须使用 PostgreSQL，可参考如下结构：*
```sql
CREATE TABLE verification_codes (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) NOT NULL,
    code VARCHAR(6) NOT NULL,
    type VARCHAR(20) NOT NULL, -- 'register' or 'reset'
    expires_at TIMESTAMP WITH TIME ZONE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
-- 建议对 (email, type) 建索引
```
