# 分镜头系统实现总结

## 概述

基于开发文档完成了分镜头脚本管理系统的完整实现，包括6张数据库表、完整的Pydantic schemas和8个RESTful API接口。

## 已完成的工作

### 1. 数据库模型 (6张表)

创建文件：`src/app/models/shot.py`

#### 表结构：

1. **script_shot_files** - 剧本文件表
   - 存储文件级别的元数据（fileId, fileName, visualStyle等）
   - 关联到外部的 library_id

2. **script_shots** - 镜头基础表
   - 存储镜头核心信息（镜号、原始文本、时长、类型等）
   - 包含场景描述、核心概念、备注字段

3. **script_characters** - 角色注册表
   - 实现角色复用逻辑
   - 同一剧本库下角色名唯一（unique constraint）

4. **script_shot_characters** - 镜头角色详情表
   - 存储角色在特定镜头中的外观和提示词
   - 支持三视图（正面、背面、侧面）的提示词和图片URL

5. **script_shot_scenes** - 镜头场景表
   - 存储场景描述、景别、运镜信息
   - 支持场景三视图

6. **script_shot_media** - 镜头媒体表
   - 存储视频生成提示词和结果
   - 存储音频信息（对白、旁白、情感、音效）

### 2. Pydantic Schemas

创建文件：`src/app/schemas/shot.py`

#### 主要Schema类：

**导入相关：**
- `ShotImportRequest` - 导入请求体（支持文档中的JSON格式）
- `ShotImportResponse` - 导入响应
- 支持驼峰命名和下划线命名的自动转换

**查询相关：**
- `ShotListResponse` - 列表响应（带分页）
- `ShotListItem` - 列表项
- `ShotDetailResponse` - 详情响应（完整数据）

**更新相关：**
- `ShotUpdateRequest` - 更新请求
- 支持部分更新（所有字段都是Optional）

**角色相关：**
- `CharacterListResponse` - 角色列表响应
- `CharacterListItem` - 角色列表项（包含出现次数）

### 3. API接口实现

创建文件：`src/app/api/v4/shot.py`

#### 已实现的8个接口：

| 方法 | 路由 | 功能 | 状态 |
|------|------|------|------|
| POST | `/api/v4/shot/import` | 导入剧本JSON | ✅ |
| GET | `/api/v4/shot/list` | 获取镜头列表（支持多种筛选） | ✅ |
| GET | `/api/v4/shot/{id}` | 获取单个镜头详情 | ✅ |
| PUT | `/api/v4/shot/{id}` | 更新镜头信息 | ✅ |
| DELETE | `/api/v4/shot/{id}` | 删除镜头 | ✅ |
| GET | `/api/v4/shot/characters` | 获取角色列表 | ✅ |

### 4. 核心功能特性

#### 角色复用逻辑
- 导入时自动检查角色是否已存在
- 同一剧本库下相同名字的角色共享同一个ID
- 不同镜头中的角色外观和提示词独立存储

#### 筛选功能
列表接口支持以下筛选条件：
- `library_id` - 剧本库ID（必填）
- `file_id` - 文件ID
- `shot_number` - 镜号
- `character_name` - 角色名（模糊搜索）
- `shot_size` - 景别
- `has_video` - 是否已生成视频
- 分页支持（page, page_size）

#### 级联删除
- 删除镜头时自动删除关联的角色详情、场景、媒体数据
- 使用数据库级别的 CASCADE 约束

#### 事务处理
- 导入接口使用事务确保数据一致性
- 失败时自动回滚

## 使用示例

### 1. 导入剧本

```bash
curl -X POST "http://localhost:7767/api/v4/shot/import" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "librarie_id": 9479678869893403,
    "fileInfo": {
      "fileId": "FILE_123",
      "fileName": "第一章.txt",
      "totalWords": 158,
      "scriptGenerationTime": "2023-10-27T10:00:00Z"
    },
    "globalConfig": {
      "visualStyle": "2D Anime"
    },
    "scriptShotList": [...]
  }'
```

### 2. 查询镜头列表

```bash
curl -X GET "http://localhost:7767/api/v4/shot/list?library_id=9479678869893403&page=1&page_size=20" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 3. 获取镜头详情

```bash
curl -X GET "http://localhost:7767/api/v4/shot/105" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 4. 更新镜头

```bash
curl -X PUT "http://localhost:7767/api/v4/shot/105" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "memo": "需要重新生成",
    "media_info": {
      "video_prompt_pos": "高清，4k，微风拂过"
    }
  }'
```

### 5. 删除镜头

```bash
curl -X DELETE "http://localhost:7767/api/v4/shot/105" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 6. 获取角色列表

```bash
curl -X GET "http://localhost:7767/api/v4/shot/characters?library_id=9479678869893403" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 技术要点

### 1. 异步数据库操作
所有数据库操作都使用 SQLAlchemy async 模式：
```python
result = await db.execute(select(ScriptShot).where(...))
await db.commit()
```

### 2. 关系映射
使用 SQLAlchemy relationship 实现表关联：
```python
shots = relationship("ScriptShot", back_populates="file", cascade="all, delete-orphan")
```

### 3. 认证保护
所有接口都需要JWT认证：
```python
current_user: User = Depends(get_current_user)
```

### 4. 响应格式统一
使用通用响应包装器：
```python
Response[T](code=200, message="Success", data=...)
```

## 注意事项

1. **角色复用**：同一剧本库下的角色会自动复用，但外观描述和提示词在每个镜头中独立存储
2. **级联删除**：删除文件会删除所有关联镜头，删除镜头会删除所有关联数据
3. **分页查询**：列表接口默认每页20条，最大100条
4. **筛选性能**：使用了数据库索引优化查询性能
5. **事务安全**：导入操作使用事务，失败会自动回滚
