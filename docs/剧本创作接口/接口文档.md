为了将这个复杂的JSON数据高效、结构化地存储到PostgreSQL，并满足“既能单独查询分镜头，又能还原完整JSON”的需求，我们需要采用**关系型结构（核心字段）+ JSONB（复杂嵌套字段）**的混合存储策略。

### 1. 核心思路

*   **拆分层级**：JSON 顶层是“文件/项目信息”和“全局配置”，属于**父级**；`script_shot_list` 是列表，属于**子级**。这是一对多的关系。
*   **扁平化与保留**：
    *   对于用于搜索、筛选、排序的字段（如 `file_id`, `shot_number`, `character_name`, `type`），必须**扁平化**为独立的表列。
    *   对于仅用于展示或结构极深且不常作为查询条件的字段（如多语言的提示词 `prompt`），使用 PostgreSQL 的 **JSONB** 类型存储。这样既保持了灵活性，又避免了创建十几张琐碎的关联表。
*   **关联方式**：通过 `project_id` (主键) 或 `file_id` (业务ID) 进行外键关联。

---

### 2. 数据库表单设计

我们将创建两张核心表：一张存项目主信息，一张存分镜头详情。

#### 表1：项目主表 (`scriptwriting_projects`)
存储 `file_info` 和 `global_config`。

```sql
CREATE TABLE scriptwriting_projects (
    id SERIAL PRIMARY KEY,                    -- 内部自增主键
    file_id VARCHAR(50) UNIQUE NOT NULL,      -- 业务ID (如 FILE_123456789)
    file_name VARCHAR(255),                   -- 文件名
    total_word_count INT,                     -- 总字数
    script_generation_time TIMESTAMP,         -- 生成时间
    visual_style VARCHAR(50),                 -- 风格 (从 global_config 提取)
    context_usage_count INT,                  -- 上下文数量 (从 global_config 提取)
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引，加快查找
CREATE INDEX idx_script_file_id ON scriptwriting_projects(file_id);
```

#### 表2：分镜头详情表 (`scriptwriting_shots`)
存储 `script_shot_list` 中的每一项。关键信息独立成列，复杂嵌套信息存 JSONB。

```sql
CREATE TABLE scriptwriting_shots (
    id SERIAL PRIMARY KEY,
    project_id INT REFERENCES scriptwriting_projects(id) ON DELETE CASCADE, -- 外键关联
    shot_number INT NOT NULL,                  -- 镜号
    
    -- 基础信息
    original_text TEXT,                        -- 原文
    type VARCHAR(50),                          -- 类型 (dialogue/action...)
    duration VARCHAR(20),                      -- 时长
    video_url TEXT,                            -- 最终视频地址
    
    -- 角色信息 (扁平化提取，方便查询某角色的所有镜头)
    character_name VARCHAR(100),
    character_gender VARCHAR(20),
    character_desc TEXT,
    
    -- 视觉信息 (扁平化常用字段)
    scene_content TEXT,
    shot_size VARCHAR(50),
    camera_movement VARCHAR(50),
    
    -- 图片地址 (扁平化)
    front_image_url TEXT,
    back_image_url TEXT,
    side_image_url TEXT,

    -- 音频信息 (扁平化)
    dialogue_content TEXT,
    voice_over TEXT,
    voice_emotion VARCHAR(50),
    sound_effects TEXT,
    
    -- 复杂嵌套数据 (JSONB)
    -- 这里存储 text_to_image_prompt, image_to_video_prompt 等复杂结构
    prompts_data JSONB, 
    
    context_summary TEXT,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引，保证查询某个文件的镜头列表速度极快
CREATE INDEX idx_shots_project_id ON scriptwriting_shots(project_id);
-- 联合索引，防止同一项目中镜号重复
CREATE UNIQUE INDEX idx_shots_project_number ON scriptwriting_shots(project_id, shot_number);
```

### 3. 存储与读取的逻辑思路

#### 存储逻辑 (Write)
1.  **解析 JSON**：后端接收到完整的 JSON 对象。
2.  **插入主表**：提取 `file_info` 和 `global_config`，插入 `scriptwriting_projects`，并获取返回的 `id` (例如 id=10)。
3.  **遍历插入子表**：遍历 `script_shot_list` 数组。
    *   将 `text_to_image_prompt` 和 `image_to_video_prompt` 组合成一个 JSON 对象放入 `prompts_data` 字段。
    *   将 `character_info.character_name` 等字段提取出来放入对应列。
    *   执行 Insert 操作，`project_id` 设为 10。

#### 读取逻辑 (Read)
1.  **完整还原 JSON**：
    *   查询 `scriptwriting_projects` 获取头部信息。
    *   查询 `scriptwriting_shots WHERE project_id = ? ORDER BY shot_number ASC`。
    *   在代码层面（Python/Go/Java）将数据库取出的扁平数据重新组装成 JSON 嵌套结构。
2.  **单个查找**：
    *   直接 `SELECT * FROM scriptwriting_shots WHERE id = ?`。

### 4. API 接口文档 (v1/scriptwriting/)

以下是标准的 RESTful API 设计。

**Base URL**: `https://api.yourdomain.com/v1/scriptwriting`

#### 4.1. 创建/导入剧本 (Store Script)
将完整的 JSON 数据存入数据库。

*   **URL**: `/projects`
*   **Method**: `POST`
*   **Content-Type**: `application/json`
*   **Body**: (即你提供的那个完整的 JSON)
*   **Response**:
    ```json
    {
      "code": 200,
      "message": "Script imported successfully",
      "data": {
        "project_id": 10,
        "file_id": "FILE_123456789",
        "shot_count": 3
      }
    }
    ```

#### 4.2. 获取完整剧本详情 (Get Full Script JSON)
通过 file_id 获取并还原成原始 JSON 结构。

*   **URL**: `/projects/{file_id}`
*   **Method**: `GET`
*   **Path Params**:
    *   `file_id`: 例如 `FILE_123456789`
*   **Response**: (返回与上传时一致的完整嵌套 JSON)

#### 4.3. 获取分镜头列表 (分页查询)
不需要一次性加载所有 prompt 详情，只列出大概。

*   **URL**: `/projects/{file_id}/shots`
*   **Method**: `GET`
*   **Query Params**:
    *   `page`: 1
    *   `page_size`: 20
*   **Response**:
    ```json
    {
      "data": [
        {
          "shot_id": 101,
          "shot_number": 1,
          "original_text": "林师兄...",
          "visual_content_summary": "一个穿着杂役服饰的弟子...",
          "status": "completed"
        }
        // ...
      ]
    }
    ```

#### 4.4. 获取单个分镜头详情 (Get Single Shot)
用于前端点击某个镜头进行编辑或查看详情。

*   **URL**: `/shots/{shot_id}`
*   **Method**: `GET`
*   **Response**:
    ```json
    {
      "shot_id": 101,
      "shot_number": 1,
      "file_id": "FILE_123456789",
      "original_text": "...",
      "character_info": { ... },
      "visual_content": { ... }, // 包含 prompt 详情
      "video": "url..."
    }
    ```

#### 4.5. 更新单个分镜头 (Update Single Shot)
例如用户修改了 Prompt 或生成了新的视频链接。

*   **URL**: `/shots/{shot_id}`
*   **Method**: `PATCH`
*   **Body**:
    ```json
    {
      "visual_content": {
        "text_to_image_prompt": {
           "CN": { "positive_prompt": "修改后的提示词..." }
        }
      },
      "video": "new_video_url.mp4"
    }
    ```

#### 4.6. 搜索接口 (Advanced Search)
例如：查找“林风眠”出现的所有镜头。

*   **URL**: `/search/shots`
*   **Method**: `POST`
*   **Body**:
    ```json
    {
      "character_name": "林风眠",
      "visual_style": "2D Anime"
    }
    ```

### 5. PostgreSQL 高级查询示例 (SQL伪代码)

如果你想直接在数据库层面组装 JSON (PostgreSQL 强项)，可以使用 `json_build_object`，这样后端代码会非常简洁：

```sql
-- 一条SQL直接查出完整的原始JSON结构
SELECT json_build_object(
    'file_info', json_build_object(
        'file_id', p.file_id,
        'file_name', p.file_name,
        'total_word_count', p.total_word_count,
        'script_generation_time', p.script_generation_time
    ),
    'global_config', json_build_object(
        'visual_style', p.visual_style,
        'context_usage_count', p.context_usage_count
    ),
    'script_shot_list', (
        SELECT json_agg(
            json_build_object(
                'shot_number', s.shot_number,
                'original_text', s.original_text,
                'type', s.type,
                'character_info', json_build_object(
                    'character_name', s.character_name,
                    'gender', s.character_gender,
                    'appearance_description', s.character_desc
                ),
                -- 这里的 prompts_data 是直接存的 JSONB，直接取出来即可
                'text_to_image_prompt', s.prompts_data->'text_to_image_prompt'
                -- 其他字段依此类推...
            ) ORDER BY s.shot_number ASC
        )
        FROM scriptwriting_shots s
        WHERE s.project_id = p.id
    )
)
FROM scriptwriting_projects p
WHERE p.file_id = 'FILE_123456789';
```
