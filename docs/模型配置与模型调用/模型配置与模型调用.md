这是一个非常典型的企业级SaaS（多租户）后台设计需求。根据你的要求，我们需要注重数据的**隔离性**、**安全性**（Key加密）、**ID的特殊格式**以及**API的规范性**（无大写、无URL参数ID）。

以下是完整的解决方案设计。

### 第一部分：数据库设计 (PostgreSQL)

我们需要一张表来存储模型配置。核心在于 `tenant_id`（租户ID）字段，它是实现逻辑隔离的关键。

**设计思路：**
1.  **ID格式**：`wm` + 20位数字。由于这是非标准UUID，建议在应用层（后端代码）生成并校验，数据库层面设为 `VARCHAR(22)`。
2.  **安全性**：`api_key` 字段必须存储加密后的字符串（如AES加密），决不能存明文。
3.  **索引**：必须在 `tenant_id` 上建立索引，保证查询效率。

```sql
-- 1. 创建模型配置表
CREATE TABLE model_configs (
    -- ID格式为 wm + 20位数字，例如 wm12345678901234567890
    id VARCHAR(22) PRIMARY KEY,
    
    -- 租户ID，用于隔离不同用户的数据（关联到你的用户/租户表）
    tenant_id VARCHAR(64) NOT NULL,
    
    -- 模型名称，如 "gpt-4", "my-custom-model"
    model_name VARCHAR(100) NOT NULL,
    
    -- 基础URL，如 "https://api.openai.com/v1" 或私有部署地址
    base_url VARCHAR(255) NOT NULL,
    
    -- 加密后的Key，应用层加密后存入，取出时解密
    encrypted_api_key TEXT NOT NULL,
    
    -- 备注信息（可选）
    description TEXT,
    
    -- 软删除标记（可选，用于数据安全）
    is_deleted BOOLEAN DEFAULT FALSE,
    
    -- 时间戳
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 2. 创建索引
-- 极其重要：为了查询隔离，所有查询都会带上 tenant_id
CREATE INDEX idx_model_configs_tenant ON model_configs(tenant_id);

-- 3. （可选）数据库层面的ID格式约束检查
ALTER TABLE model_configs 
ADD CONSTRAINT check_id_format 
CHECK (id ~ '^wm\d{20}$');
```

### 第二部分：API 接口文档设计

**设计原则：**
1.  **Base URL**: `/api/v2/model_config/`
2.  **鉴权**: 所有接口必须在Header中携带 Token，后端中间件解析出 `current_tenant_id`。
3.  **参数**: 全小写下划线风格 (`snake_case`)，无大写。
4.  **风格**: RESTful 风格，使用标准 HTTP 方法（GET、POST、PUT、DELETE）。

#### 1. 创建模型配置
*   **URL**: `POST /api/v2/model_config/create`
*   **描述**: 新增一个模型配置。
*   **请求体 (JSON)**:
    ```json
    {
      "model_name": "gpt-4-turbo",
      "base_url": "https://api.openai.com/v1",
      "api_key": "sk-xxxxxxxx",
      "description": "财务团队专用"
    }
    ```
*   **响应 (JSON)**:
    ```json
    {
      "code": 200,
      "msg": "success",
      "data": {
        "config_id": "wm20231027000000000001",
        "model_name": "gpt-4-turbo",
        "base_url": "https://api.openai.com/v1"
      }
    }
    ```

#### 2. 查询列表 (带分页)
*   **URL**: `GET /api/v2/model_config/list`
*   **描述**: 获取当前租户下的所有配置（严禁返回解密后的Key）。
*   **查询参数**:
    - `page`: 页码（默认 1）
    - `page_size`: 每页数量（默认 10）
    - `keyword`: 可选，按名称搜索（支持中文，需 URL 编码）
*   **示例**: `GET /api/v2/model_config/list?page=1&page_size=10&keyword=gpt`
*   **响应 (JSON)**:
    ```json
    {
      "code": 200,
      "msg": "success",
      "data": {
        "total": 5,
        "list": [
          {
            "config_id": "wm20231027000000000001",
            "model_name": "gpt-4-turbo",
            "base_url": "https://api.openai.com/v1",
            "description": "财务团队专用",
            "created_at": "2023-10-27T10:00:00+00:00"
          }
        ]
      }
    }
    ```

#### 3. 更新配置
*   **URL**: `PUT /api/v2/model_config/{config_id}`
*   **描述**: 修改配置。必须校验该 `config_id` 是否属于当前 `tenant_id`。
*   **路径参数**: `config_id` - 配置ID（wm + 20位数字）
*   **请求体 (JSON)**:
    ```json
    {
      "model_name": "gpt-4-turbo-preview",
      "api_key": "sk-new-key-xxxxx",
      "base_url": "https://new-url.com",
      "description": "更新后的描述"
    }
    ```
    注意：所有字段均为可选，只传需要更新的字段。
*   **响应**:
    ```json
    { "code": 200, "msg": "updated successfully" }
    ```

#### 4. 删除配置
*   **URL**: `DELETE /api/v2/model_config/{config_id}`
*   **描述**: 软删除指定配置。
*   **路径参数**: `config_id` - 配置ID（wm + 20位数字）
*   **响应**:
    ```json
    { "code": 200, "msg": "deleted successfully" }
    ```

#### 状态码说明

| 状态码 (Code) | HTTP Status | 含义 | 说明 |
| :--- | :--- | :--- | :--- |
| **200** | 200 | OK | 操作成功 |
| **400** | 400 | Bad Request | 参数错误（如缺少必填项、含有大写字母等） |
| **401** | 401 | Unauthorized | 未登录或Token过期 |
| **403** | 403 | Forbidden | **越权访问**（试图操作不属于该租户ID的配置） |

| **500** | 500 | Internal Server Error | 服务器内部错误（数据库连接失败、加密失败等） |

### 第三部分：代码实现逻辑与调用 (Python + LangChain)

这里包含两部分逻辑：
1.  **ID生成工具**：满足 `wm` + 20位数字。
2.  **模型调用工厂**：根据配置ID动态加载 LangChain 模型。

假设你使用的是 Python (FastAPI/Flask) 作为后端。

#### 1. 辅助工具 (ID生成与加解密)

```python
import random
import string
from cryptography.fernet import Fernet # 需要安装 cryptography

# 模拟一个密钥，实际应从环境变量读取
ENCRYPTION_KEY = Fernet.generate_key() 
cipher_suite = Fernet(ENCRYPTION_KEY)

def generate_wm_id():
    """生成 wm + 20位数字 的ID"""
    # 生成20位随机数字字符串
    digits = ''.join(random.choices(string.digits, k=20))
    return f"wm{digits}"

def encrypt_key(raw_key: str) -> str:
    """加密API Key"""
    return cipher_suite.encrypt(raw_key.encode()).decode()

def decrypt_key(encrypted_key: str) -> str:
    """解密API Key"""
    return cipher_suite.decrypt(encrypted_key.encode()).decode()
```

#### 2. LangChain 调用逻辑实现

这是一个如何使用上述配置来初始化 `ChatOpenAI` 的服务层代码示例。

```python
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage
# 假设这是你的数据库连接对象
import db_driver 

class ModelFactory:
    def __init__(self, tenant_id: str):
        self.tenant_id = tenant_id # 这里是有团队的情况那如果没有团队呢

    def get_chat_model(self, config_id: str):
        """
        根据配置ID获取LangChain ChatOpenAI实例
        """
        # 1. 查询数据库
        # SQL逻辑: SELECT * FROM model_configs WHERE id = :config_id AND tenant_id = :tenant_id
        # 必须加上 tenant_id 条件，防止B租户使用了A租户的模型ID
        config = db_driver.fetch_one(
            "SELECT * FROM model_configs WHERE id = %s AND tenant_id = %s", 
            (config_id, self.tenant_id)
        )
        
        if not config:
            raise ValueError("Configuration not found or permission denied")
        
        # 2. 解密 Key
        real_api_key = decrypt_key(config['encrypted_api_key'])
        
        # 3. 初始化 LangChain 模型
        chat_model = ChatOpenAI(
            openai_api_key=real_api_key,
            openai_api_base=config['base_url'],
            model_name=config['model_name'],
            temperature=0.7
        )
        
        return chat_model

# ================= 使用示例 =================

def chat_with_tenant_model(tenant_id, config_id, user_input):
    try:
        # 1. 实例化工厂
        factory = ModelFactory(tenant_id="tenant_A_001")
        
        # 2. 获取配置好的模型 (传入 wm...ID)
        llm = factory.get_chat_model(config_id="wm12345678901234567890")
        
        # 3. 调用模型对话
        messages = [HumanMessage(content=user_input)]
        response = llm.invoke(messages)
        
        return response.content
        
    except Exception as e:
        return f"Error: {str(e)}"

# 模拟调用
# print(chat_with_tenant_model("tenant_A_001", "wm12345678901234567890", "你好，你是哪个模型？"))
```

### 总结要点

1.  **隔离实现**：SQL 语句中永远带上 `AND tenant_id = ?`，这是物理表单共享但逻辑隔离的铁律。
2.  **路由规范**：完全遵守了不使用 URL ID 参数，所有敏感或定位参数均放入 Body，且 Key 使用全小写。
3.  **安全性**：Key 在落库前被 `encrypt_key` 加密，在使用前由 `decrypt_key` 解密，数据库管理员也无法直接看到明文 Key。
4.  **ID规范**：在代码层通过 `generate_wm_id` 严格控制格式。
