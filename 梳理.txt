# API接口梳理

## 1. 认证相关接口 (auth.py)

### 1.1 发送验证码
- **URL**: `/api/v1/send-code`
- **方法**: POST
- **请求体**: 
  ```json
  {
    "email": "string"  # 必填，邮箱地址
  }
  ```
- **响应**: 200 OK
  ```json
  {
    "message": "Verification code sent"
  }
  ```

### 1.2 用户注册
- **URL**: `/api/v1/register`
- **方法**: POST
- **请求体**: 
  ```json
  {
    "email": "string",      # 必填，邮箱地址
    "password": "string",   # 必填，密码
    "code": "string"        # 必填，验证码
  }
  ```
- **响应**: 200 OK
  ```json
  {
    "id": "string",
    "email": "string",
    "is_active": true
  }
  ```

### 1.3 重置密码
- **URL**: `/api/v1/reset-password`
- **方法**: POST
- **请求体**: 
  ```json
  {
    "email": "string",        # 必填，邮箱地址
    "code": "string",         # 必填，验证码
    "new_password": "string"  # 必填，新密码
  }
  ```
- **响应**: 200 OK
  ```json
  {
    "message": "Password reset successfully"
  }
  ```

### 1.4 用户登录
- **URL**: `/api/v1/login`
- **方法**: POST
- **请求体**: 
  ```json
  {
    "email": "string",      # 必填，邮箱地址
    "password": "string"   # 必填，密码
  }
  ```
- **响应**: 200 OK
  ```json
  {
    "access_token": "string",
    "token_type": "bearer",
    "user_id": "string"
  }
  ```

### 1.5 获取当前用户信息
- **URL**: `/api/v1/me`
- **方法**: GET
- **认证**: Bearer Token
- **响应**: 200 OK
  ```json
  {
    "id": "string",
    "email": "string",
    "is_active": true
  }
  ```

## 2. LLM对话接口 (llm.py)

### 2.1 与LLM对话
- **URL**: `/api/v1/chat`
- **方法**: POST
- **请求体**: 
  ```json
  {
    "message": "string",                # 必填，用户消息
    "use_storyboard_prompt": false,      # 可选，是否使用分镜头脚本生成提示词
    "custom_prompt_file": "string"      # 可选，自定义提示词文件名
  }
  ```
- **响应**: 200 OK
  ```json
  {
    "reply": "string"
  }
  ```

## 3. 媒体生成接口 (media.py)

### 3.1 生成图片
- **URL**: `/api/v1/generate/image`
- **方法**: POST
- **认证**: Bearer Token
- **请求体**: 
  ```json
  {
    "prompt": "string",        # 必填，生成提示词
    "width": 1024,             # 可选，图片宽度
    "height": 1024,            # 可选，图片高度
    "style": "string"          # 可选，图片风格
  }
  ```
- **响应**: 200 OK 或 501 Not Implemented

### 3.2 生成视频
- **URL**: `/api/v1/generate/video`
- **方法**: POST
- **认证**: Bearer Token
- **请求体**: 
  ```json
  {
    "prompt": "string",        # 必填，生成提示词
    "duration": 5,             # 可选，视频时长（秒）
    "style": "string"          # 可选，视频风格
  }
  ```
- **响应**: 200 OK 或 501 Not Implemented

## 4. 脚本库管理接口 (script.py)

### 4.1 创建脚本库
- **URL**: `/api/v1/libraries`
- **方法**: POST
- **认证**: Bearer Token
- **请求体**: 
  ```json
  {
    "name": "string",          # 必填，库名称
    "type": "string",          # 必填，库类型
    "description": "string"     # 可选，库描述
  }
  ```
- **响应**: 200 OK
  ```json
  {
    "id": 1234567890123456,
    "user_id": "string",
    "name": "string",
    "type": "string",
    "description": "string",
    "minio_folder_path": "string",
    "created_at": "string",
    "updated_at": "string"
  }
  ```

### 4.2 获取脚本库列表
- **URL**: `/api/v1/libraries`
- **方法**: GET
- **认证**: Bearer Token
- **响应**: 200 OK
  ```json
  [
    {
      "id": 1234567890123456,
      "user_id": "string",
      "name": "string",
      "type": "string",
      "description": "string",
      "minio_folder_path": "string",
      "created_at": "string",
      "updated_at": "string"
    }
  ]
  ```

### 4.3 获取脚本库详情
- **URL**: `/api/v1/libraries/{lib_id}`
- **方法**: GET
- **认证**: Bearer Token
- **路径参数**: 
  - `lib_id`: 脚本库ID（必填）
- **响应**: 200 OK
  ```json
  {
    "id": 1234567890123456,
    "user_id": "string",
    "name": "string",
    "type": "string",
    "description": "string",
    "minio_folder_path": "string",
    "created_at": "string",
    "updated_at": "string",
    "files": [
      {
        "id": 123456789012345678,
        "library_id": 1234567890123456,
        "filename": "string",
        "file_url": "string",
        "minio_object_key": "string",
        "file_type": "string",
        "created_at": "string",
        "updated_at": "string"
      }
    ]
  }
  ```

### 4.4 删除脚本库
- **URL**: `/api/v1/libraries/{lib_id}`
- **方法**: DELETE
- **认证**: Bearer Token
- **路径参数**: 
  - `lib_id`: 脚本库ID（必填）
- **响应**: 200 OK
  ```json
  {
    "msg": "Library and associated files deleted"
  }
  ```

### 4.5 上传文件到脚本库
- **URL**: `/api/v1/libraries/{lib_id}/files`
- **方法**: POST
- **认证**: Bearer Token
- **路径参数**: 
  - `lib_id`: 脚本库ID（必填）
- **请求体**: Form Data
  - `file`: 文件（必填）
- **响应**: 200 OK
  ```json
  {
    "id": 123456789012345678,
    "library_id": 1234567890123456,
    "filename": "string",
    "file_url": "string",
    "minio_object_key": "string",
    "file_type": "string",
    "created_at": "string",
    "updated_at": "string"
  }
  ```

### 4.6 获取脚本库文件列表
- **URL**: `/api/v1/libraries/{lib_id}/files`
- **方法**: GET
- **认证**: Bearer Token
- **路径参数**: 
  - `lib_id`: 脚本库ID（必填）
- **查询参数**: 
  - `file_type`: 可选，文件类型过滤（video/audio/image/text）
- **响应**: 200 OK
  ```json
  [
    {
      "id": 123456789012345678,
      "library_id": 1234567890123456,
      "filename": "string",
      "file_url": "string",
      "minio_object_key": "string",
      "file_type": "string",
      "created_at": "string",
      "updated_at": "string"
    }
  ]
  ```

### 4.7 删除文件
- **URL**: `/api/v1/files/{file_id}`
- **方法**: DELETE
- **认证**: Bearer Token
- **路径参数**: 
  - `file_id`: 文件ID（必填）
- **响应**: 200 OK
  ```json
  {
    "msg": "File deleted"
  }
  ```

### 4.8 获取文件内容
- **URL**: `/api/v1/files/{file_id}/content`
- **方法**: GET
- **认证**: Bearer Token
- **路径参数**: 
  - `file_id`: 文件ID（必填）
- **查询参数**: 
  - `max_length`: 可选，文本文件最大返回长度
- **响应**: 200 OK（返回文件内容，Content-Type根据文件类型自动设置）

# 技术栈

- **后端框架**: FastAPI
- **数据库**: SQLAlchemy (异步)
- **认证**: JWT
- **对象存储**: MinIO
- **AI服务**: 自定义LLM服务、AI媒体生成服务
- **邮件服务**: 自定义邮件服务

# 项目特点

1. **模块化设计**: 清晰的目录结构，便于维护和扩展
2. **异步编程**: 使用FastAPI和SQLAlchemy异步模式，提高性能
3. **完整的认证体系**: 支持注册、登录、密码重置等功能
4. **脚本库管理**: 支持多种类型文件的上传、管理和访问
5. **脚本写作功能**: 支持分镜脚本的创建、编辑和搜索
6. **AI集成**: 集成LLM对话和AI媒体生成功能
7. **安全设计**: 密码加密、验证码验证、JWT认证

# 启动流程

1. 安装依赖: `pip install -r requirements.txt`
2. 配置环境变量: 复制 `.env.example` 为 `.env` 并填写相关配置
3. 启动应用: `python main.py`
4. 访问API文档: `http://localhost:8000/docs`

# 主要功能模块

1. **用户管理**: 注册、登录、密码重置
2. **脚本库**: 文件上传、管理、访问
3. **脚本写作**: 分镜脚本的创建、编辑、搜索
4. **AI服务**: LLM对话、AI媒体生成
5. **媒体管理**: 图片、视频等媒体资源的管理
